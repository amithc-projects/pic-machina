# AI Assisted Features in InSPyReNet

Roadmap for features that can reuse the InSPyReNet SwinB model shipped in
[PR #6](https://github.com/amithc-projects/pic-machina/pull/6). The model
produces a 0–1 saliency matte — "where is the subject?" — which is a
reusable primitive for far more than background removal.

## Shared Refactor (prerequisite) — ✅ Shipped in #7

Every feature below wants the **raw saliency matte**, not the finished "alpha
channel replaced" canvas. One small refactor upfront makes the rest cheap:

**In `app/src/engine/ai/inspyrenet.js`:**

1. Split `runInspyrenet` into two exports:
   - `getSaliencyMask(canvas, { size? }) → { mask: Uint8ClampedArray, width, height }`
   - `applyMaskAsAlpha(ctx, mask, { mode, edgeSmoothing })` (the current alpha
     write-back logic)
2. Add an in-memory **last-mask cache** keyed by a cheap canvas signature
   (dimensions + corner pixel sample + hash of a 16×16 downsample).
   Invalidate on dimension change or signature mismatch.
3. When multiple saliency-using transforms run in sequence on the same
   image, inference happens **once** per image instead of once per transform.

This turns every feature below from "+1–2 s per transform" into "+1–2 s per
image, no matter how many saliency transforms are stacked."

Also expose on the asset record (via `patchAsset(hash, { vision: {...} })`):
- `vision.subjectBBox` — `{ x, y, w, h }` in source pixels
- `vision.subjectArea` — % of frame above threshold
- `vision.subjectCentroid` — `{ x, y }`
- `vision.matteAt` — timestamp

So batch re-runs with different params read the bbox back instead of
re-running inference.

## 1. Subject-Aware Cropping — `ai-subject-crop` — ✅ Shipped in #8

**Module:** `app/src/engine/transforms/geometry.js`

Crops to a target aspect ratio while keeping the subject composed correctly.
Fills the gap between `geo-face-crop` (people only) and the `smartcrop`
heuristic dep (no semantic understanding).

**Params:**
| name | type | default | notes |
|---|---|---|---|
| aspectRatio | select | `original` | `1:1`, `4:5`, `3:4`, `4:3`, `16:9`, `9:16`, `custom` |
| customRatio | text | `1.618` | shown when `aspectRatio=custom` |
| padding | range 0–50 | 10 | % of subject bbox, not image |
| anchor | select | `center` | `center`, `thirds-tl/tr/bl/br`, `top`, `bottom` |
| zoomMode | select | `fit` | `fit` / `fill` / `pad` |
| threshold | range 10–90 | 50 | matte confidence % for bbox |

**Algorithm:**
1. Get matte → compute bbox from `mask[i] > threshold`.
2. Inflate bbox by `padding`% of its own dimensions.
3. Fit target aspect: largest rectangle of requested ratio containing the
   inflated bbox, clamped to image bounds.
4. Slide rectangle so subject centroid lands on `anchor` point.
5. Apply crop; preserve source resolution unless `zoomMode=fit` downscales.

**Graceful fallback:** if model is not downloaded, log `console.info` and
use existing `smartcrop` behavior. Document-type assets should stay on
`smartcrop` (saliency is trained on photographic subjects).

**Unlocks:**
- Content-aware thumbnails (§2) are `ai-subject-crop` with fixed params.
- Template aggregator gets per-slot smart fitting.
- Social-media batch recipes: 1:1 + 9:16 + 16:9 from one source, all three
  share the cached bbox.

## 2. Content-Aware Thumbnails

**Module:** new `app/src/utils/thumbnails.js`

Replaces center-crop thumbnails in Library, History, Showcase with crops
centered on the detected subject.

**Design:**
- `generateSmartThumbnail(canvas, { size, padding })` → calls
  `ai-subject-crop` internally with `aspectRatio='1:1'`, `anchor='center'`.
- Opt-in via a new setting `smartThumbnails: true` in `utils/settings.js`.
- **Progressive render pattern:** never block the grid. Render the existing
  center-cropped thumbnail first, then replace in-place when the smart
  version is ready. Avoids blanking the Library while 200 inferences queue.
- Cache the result in the existing thumbnail data-URL field on the asset /
  run / showcase record. Second render of the same item is free.

**Gotcha:** Bulk recompute for the existing corpus should be explicit
("Rebuild thumbnails" button), not automatic on feature-flag flip.

## 3. Portrait Bokeh — `ai-portrait-bokeh` — ✅ Shipped in #7

**Module:** `app/src/engine/transforms/ai.js`

Large-aperture lens simulation: subject sharp, background blurred, with
optional graduated depth falloff.

**Params:**
| name | type | default |
|---|---|---|
| blurRadius | range 0–60 | 15 |
| edgeFeather | range 0–30 | 8 |
| falloff | select `flat`/`graduated` | `flat` |
| apertureShape | select `gaussian`/`disk` | `gaussian` |

**Algorithm:**
1. Saliency → matte.
2. Gaussian-blur the matte by `edgeFeather` px (depth transition zone).
3. Blur a copy of the full image by `blurRadius` px via `photon.gaussian_blur`
   (already loaded by `ensurePhoton()` in `ai.js`).
4. Per-pixel composite: `out = matte * sharp + (1 - matte) * blurred`.
5. If `falloff=graduated`, blend in a second larger-radius blur based on
   distance from the subject (approximate via second large matte blur).

**Why cheap:** Photon + `gaussian_blur` are already wired up for the Face
Privacy transform. Bokeh is basically inverted-mask + larger radius.

## 4. Subject Vignette — `ai-subject-vignette`

**Module:** `app/src/engine/transforms/ai.js`

Darkens away from the subject regardless of subject position — off-center
compositions stay intentional instead of looking lopsided.

**Params:**
| name | type | default |
|---|---|---|
| strength | range 0–100 | 60 |
| softness | range 0–100 | 70 |
| color | color | `#000000` |

**Algorithm:**
1. Saliency → matte.
2. Large-radius blur of the matte (approximates Euclidean distance from
   subject).
3. `darkenAmount = (1 - blurredMatte) * strength`; lerp pixel toward `color`.

Pairs with Portrait Bokeh as a system recipe: **Cinematic Portrait** =
Bokeh + Subject Vignette + mild S-curve. Add to
`app/src/data/system-recipes.js` once both ship.

## 5. Selective Color Grade — `ai-selective-grade`

**Module:** `app/src/engine/transforms/color.js`

Separate color adjustments for subject and background: desaturated
background + vivid subject, warm/cool split, B&W background + color
subject, etc.

**Params:**
| scope | name | type | default |
|---|---|---|---|
| subject | saturation | range −100..100 | 0 |
| subject | temperature | range −100..100 | 0 |
| subject | exposure | range −100..100 | 0 |
| background | saturation | range −100..100 | −60 |
| background | temperature | range −100..100 | 0 |
| background | exposure | range −100..100 | −20 |
| shared | edgeFeather | range 0..30 | 6 |

**Algorithm:**
1. Saliency → feathered matte.
2. Create two graded copies of the whole image (subject grade, bg grade)
   using existing Photon color primitives that `color.js` already uses.
3. Per-pixel: `out = matte * subjectGraded + (1 - matte) * bgGraded`.

All color primitives exist. This transform is glue — the value is exposing
"selective color" as a single-transform step instead of a multi-node recipe.

## 6. Subject-Only Sharpen / Denoise — `ai-subject-sharpen`

**Module:** `app/src/engine/transforms/ai.js`

Sharpens the subject without amplifying noise in background bokeh, skies,
or fabric.

**Params:**
| name | type | default |
|---|---|---|
| amount | range 0–200 | 80 |
| radius | range 0.5–5 | 1.5 |
| edgeFeather | range 0–20 | 4 |
| mode | select `sharpen` / `denoise` / `both` | `sharpen` |
| inverse | checkbox | false |

**Algorithm:**
1. Saliency → feathered matte.
2. Sharpened copy (Photon `unsharpen`).
3. Optionally denoised copy (Photon noise reduction / bilateral blur).
4. Composite with matte; `inverse=true` flips subject and background roles
   (useful for product photos where you want crisp label text behind a
   blurred foreground hand).

## 7. Compositing Effects

Smaller additions that reuse the same matte primitive.

### 7a. Drop Shadow — `ai-drop-shadow` — ✅ Shipped in #9
- Blur the matte, offset it (`dx`, `dy`), tint black at alpha.
- Composite under the subject.
- Params: `offsetX`, `offsetY`, `blur`, `opacity`, `color`.

### 7b. Sticker Outline — `ai-sticker-outline` — ✅ Shipped in #9
- Dilate matte → subtract original matte → fill white.
- Classic cut-out sticker look; also a good input for social stickers.
- Params: `thickness`, `color`, `secondOutlineColor` (optional double border).

### 7c. Subject Glow — `ai-subject-glow`
- Gaussian-blur matte outward, tint, composite behind subject.
- Pairs with existing Glowing Eyes effect for full-body neon looks.
- Params: `spread`, `intensity`, `color`.

### 7d. Matte Export — `ai-export-matte`
- Outputs the raw single-channel matte as an alpha PNG.
- For Photoshop / After Effects / DCC workflows where users want to
  take the alpha elsewhere and do their own compositing.

## 8. Photo Stack & Template Aggregator — subject isolation

**Modules:**
- `app/src/engine/compositor.js`
- `app/src/engine/transforms/flow.js`

`flow-photo-stack`, `flow-animate-stack`, `flow-template-aggregator`
currently composite the full rectangular bitmap of each source. Add a
per-layer boolean **`isolateSubject`** that applies the matte as alpha
before compositing.

No batch-worker changes needed — these already live in
`MAIN_THREAD_TRANSFORMS` in `app/src/engine/batch.js`.

**Gotcha:** N images = N inference runs. Offer a **"Isolate first frame
only"** mode for video-wall / animate-stack so motion sequences don't cost
N×inference time.

## 9. Metadata & Indexing

Passive feature that compounds over time. Every AI-crop or saliency pass
stores:

```js
patchAsset(hash, {
  vision: {
    subjectBBox:     { x, y, w, h },
    subjectArea:     0.34,          // % of frame
    subjectCentroid: { x, y },
    matteAt:         Date.now()
  }
});
```

Enables History / Library filters like:
- "Close-ups" (`subjectArea > 0.5`)
- "Wide shots" (`subjectArea < 0.15`)
- "Subject-on-left" compositions (centroid x < 0.4)

No UI needed upfront — just write the field. The filter UI can come later
once enough assets are indexed.

## Recommended Build Order

1. **Shared refactor** — split `runInspyrenet`, add mask caching, persist
   `vision.subjectBBox` via `patchAsset`. Unblocks everything.
2. **Portrait Bokeh** — most visually dramatic, reuses existing Photon
   code, validates the caching pattern.
3. **Subject-Aware Cropping** — highest utility; unblocks Content-Aware
   Thumbnails and Template Aggregator improvements.
4. **Drop Shadow + Sticker Outline** — small, delightful, share the
   primitive.
5. **Selective Color Grade + Subject Vignette + Sharpen** — round out the
   photographic suite; ship a **Cinematic Portrait** system recipe that
   composes them.
6. **Template aggregator `isolateSubject`** — once subject-crop is in,
   wire it into the multi-image flow transforms.
7. **Content-Aware Thumbnails (opt-in)** — safe to ship last; depends on
   the bbox cache being reliable.

## Non-Goals

- Instance segmentation (multiple distinct subjects with separate masks) —
  InSPyReNet returns one saliency map. Would need a different model
  (e.g. SAM).
- Depth estimation — saliency is not depth. "Graduated bokeh" is an
  approximation, not a true depth map.
- Real-time preview during slider drag — ~1–2 s inference is too slow.
  Debounce to "on release" or cache mask during a session.
