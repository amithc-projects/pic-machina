What would be the use case for a loop node ?


In the context of a batch image processor like Pro Studio Batch, a Loop Node would be used for operations that need to be repeated with incremental changes or applied to multiple related sub-assets.

Here are the primary use cases:

Iterative Refinement (Recursive Processing):

Multi-pass AI Upscaling: Running a "Super Resolution" node multiple times to reach a specific target resolution (e.g., upscaling 2x, then 2x again).
Recursive Blurring/Sharpening: Applying a subtle blur or sharpen effect over several iterations to achieve a specific "softness" or "edge" that a single heavy pass can't replicate without artifacts.
Generating Variations (Parameter Sweeps):

Exposure/LUT Bracketing: A loop could run a single image through a "Color Tint" or "Exposure" node 5 times, incrementing the value by +0.5 EV each time, to generate a series of exposures for HDR or selection.
Style Exploration: Iterating through a list of different LUTs (Look-Up Tables) to output 10 different "looks" for the same image automatically.
Handling Multi-Layer or Multi-Page Assets:

PDF/TIFF Splitting: If the input is a multi-page TIFF or PDF, a loop node would iterate through each page, applying the transformation recipe to every page individually before exporting.
Video Frame Processing: For short GIFs or video clips, a loop node treats each frame as an individual image, applying the recipe to frame 1, then frame 2, and so on.
Complex Compositing (Collages/Grids):

Contact Sheet Generation: A loop could iterate through a selection of images to place them into a specific grid layout (e.g., "Repeat this layout for every 4 images selected").
In terms of the UI: A loop node would typically be visualized as a "container" or a "wrapped" section of the timeline in your Block Builder [BKB], with a defined 'Index' variable (e.g., {{loop.index}}) that can be used to dynamically change settings in each pass.
